## Design and implementation plan

1. Establish a minimal base commit, then deliver S0 through `feature/s0-foundation` and a PR.
2. Vendor the exact Studio 1.0.0 bundle, force LF, and verify hashes plus the pinned provider CLI before application checks.
3. Build a standard-library HTTP shell with explicit methods, timeouts, security headers, semantic landmarks, skip navigation, and visible focus.
4. Prove RED with missing verifier/server symbols; implement GREEN; refactor with tamper/extra-file rejection and real headless-browser checks.
5. Require contract, formatting, vet, race/unit, build, vulnerability, dependency-review, CodeQL, and Go browser checks in CI.
6. Protect `main`, merge only green CI, then update the cursor.

## Security consequences

The S0 shell has no authentication state and writes nothing to browser storage. Future OIDC tokens must remain in bounded in-memory server sessions. The browser receives only a hardened session cookie, never a provider bearer token or trusted tenant/user header.
